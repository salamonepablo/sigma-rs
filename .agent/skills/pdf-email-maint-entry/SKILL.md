# Skill: pdf-email-maint-entry

## Intent

Ensure consistent maintenance-entry output across PDF and email.

## Apply when

- Editing PDF layout/content in maintenance entry.
- Editing email subject/body/signature for maintenance entry.
- Updating dynamic labels by unit type/brand.
## Rules

- Reuse shared display rules when possible.
- Keep labels consistent between UI, PDF, and email.
- Preserve formatting conventions (dates, km separators, language).
- Avoid duplicated logic across layers.
## Checklist

- [ ] Dynamic labels resolved from shared policy/service.
- [ ] Unit-type/brand edge cases covered (e.g., Nohab, railcar classes).
- [ ] PDF layout changes do not break content flow.
- [ ] Email content remains concise and complete.
## Done criteria

- PDF and email show coherent data and labels.
- No duplicated rule divergence introduced.
