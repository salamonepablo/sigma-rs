## Resumen de Sesión - SIGMA-RS

### Contexto del Proyecto
**SIGMA-RS** es un sistema de gestión de tickets de averías para Material Rodante diésel (Línea Roca, Argentina). Stack: Django 5.1 + Waitress + SQLite + WhiteNoise.

### Lo que hicimos

1. **Estructura Clean Architecture** - Creamos la estructura de carpetas para monolito modular:
   ```
   apps/tickets/
   ├── domain/entities/, repositories/, services/
   ├── application/use_cases/
   ├── infrastructure/models/, repositories/, migrations/
   └── presentation/views/, forms/, templates/
   shared/domain/, infrastructure/
   tests/tickets/ (estructura espejo)
   ```

2. **Documentación** - Creamos/actualizamos:
   - `docs/ARCHITECTURE.md` - Documentación de capas y dependencias
   - `docs/CHANGELOG.md` - Registro de cambios
   - `AGENTS.md` - Agregamos glosario de dominio (Loc, CCRR, CM, GOP, OT) y jerarquía de unidades

3. **Diseño del Modelo de Datos** - Definimos el esquema (aún no implementado):

   **Jerarquía de Unidades de Mantenimiento:**
   ```
   UnidadMantenimiento (abstracta)
   ├── Locomotora (numero, marca_id, modelo_id, activo)
   ├── CocheRemolcado (numero, marca_id, tipo_coche_id, activo)
   └── CocheMotor (numero, marca_id, modelo_id, conformacion_id, activo)
   ```

   **Tablas Auxiliares:**
   - `Marca` - GM, Dalian CNR, Materfer, CNR
   - `Modelo` - Para Locs y CM (8G, 8H, GT22CW, GT22CW-2, etc.)
   - `TipoCoche` - Para CCRR (U=Única, FU=Furgón Única, CPA, CRA, etc.)
   - `Conformacion` - Para CM (CM-CM, CM-R-CM, etc.)

   **Ejemplos de numeración:**
   - Locomotora: A904 (GM/GT22CW), CKD 8G-0001 (CNR/8G)
   - Coche Remolcado: U3001 (Materfer/Única), CPA001 (CNR/CPA)
   - Coche Motor: (pendiente definir)
